Manual: Formatting

Time
{time:%H_%M_%S_%a_%d} - will create a string containing:
  • %H - hour
  • %M - minute
  • %S - second
  • %a - abbreviated weekday name (Sun, Mon, ...)
  • %d - day of the month (01, 02, 03, ...)
For more details on the available time formats: link
Other
  • {timestamp} - timestamp in milliseconds
  • {capture_index}
  • {camera_id}
  • {camera_serial_nb}
  • {camera_name}
  • {timelapse_name}
  • {timelapse_id} - internal unique autogenerated ID assigned to a timelapse
  • {x:.2} - to truncate a value. Example: {camera_name:.2} - will give only the first 2 letters of the camera's name.
General examples
  • ~/Desktop/Timelapses/{timelapse_name}_{timelapse_id}/capture_{capture_index}
    To have a separate folder for each timelapse and a separate folder for each capture.
  • Timelapses/{time:%b}/{time:%A}
    To group the images by months, then by days. (e.g. Jan/Monday/)
  • Timelapses/{camera_serial_nb}
    To have a separate folder for each device. The serial number is read from the camera, so it should be unique.
  • capture_{capture_index}
    File names containing the capture index: capture_0.nef, capture_1.nef, ...
Note
The files will be overwritten if formatting is not used. For example: having the Storage dir format: Timelapses and the Filename format: capture_{capture_index}, will result in overwriting the same file if we had multiple cameras.

Why?
Let's say that we had 2 cameras: Camera1 and Camera2. And capture_index = 1.
The filename would be something similar to: capture_1.nef (or capture_1.jpg but that's not relevant now). So, both cameras would try to write to the same file. So, instead of having 2 images for each camera, we would end up with only 1.

Quick solution:
We could simply add the camera's id or its serial number to the name: capture_{camera_serial_nb}_{capture_index}.
